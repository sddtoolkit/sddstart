# 版本号规范

## 元信息
- 版本：v1.0
- 生效日期：2026-03-05
- 最后更新：2026-03-05
- 责任人：reviewer-agent
- 变更说明：初始版本，定义 Agents/Skills/Tools 及 SDD 核心文档的版本号规则。

## 目标与范围
- 目标：建立统一的版本号规则，确保 SDD 文档体系的版本可追溯、兼容性可判断、变更影响可评估。
- 适用范围：
  - Agents 定义文档（`specs/meta/agents/`）
  - Skills 定义文档（`specs/meta/skills/`）
  - Tools 适配与工具链（`specs/meta/tools/`、`specs/meta/index/`）
  - 调度契约文件（`agent-dispatch.json`、`tool-adapters.json`）
  - 治理与标准文档（`specs/govs/`、`specs/standards/`）

## 版本号格式

### 基本格式
```
v<MAJOR>.<MINOR>[-<STAGE>]
```

| 字段 | 说明 | 示例 |
|------|------|------|
| MAJOR | 主版本号，不兼容的结构性变更 | `v1`, `v2` |
| MINOR | 次版本号，向后兼容的功能增强或内容补充 | `v1.1`, `v1.4` |
| STAGE | 可选阶段标识，用于预发布或草案 | `v1.5-draft`, `v2.0-rc1` |

### 阶段标识规范

| 标识 | 含义 | 使用场景 |
|------|------|----------|
| `-draft` | 草案 | 初始编写，尚未评审 |
| `-review` | 评审中 | 已提交评审，待确认 |
| `-rc<N>` | 发布候选 | 第 N 轮候选版本，通过后转为正式版 |
| `-beta` | 试用版 | 小范围试用，收集反馈 |
| `-deprecated` | 废弃 | 标记即将移除，保留过渡期 |

## 分层版本规则

### 1. Agents 版本规则

**定义位置**：`specs/meta/agents/agents.md` 及各 Agent 细则文件

**版本递增规则**：

| 变更类型 | 版本动作 | 示例 |
|----------|----------|------|
| 新增 Agent 角色 | MINOR++ | `v1.3` → `v1.4` |
| 删除 Agent 角色 | MAJOR++ | `v1.x` → `v2.0` |
| 修改 Agent 职责边界 | MINOR++ | `v1.2` → `v1.3` |
| 调整 Agent-Skill 映射 | MINOR++ | `v1.1` → `v1.2` |
| 修改共享 Skill 仲裁规则 | MAJOR++（若影响流程阻断）或 MINOR++ | 视影响范围 |
| 补充说明性文字 | MINOR++ | `v1.0` → `v1.1` |

**元信息必填字段**：
```markdown
## 元信息
- 版本：v<MAJOR>.<MINOR>
- 生效日期：YYYY-MM-DD
- 最后更新：YYYY-MM-DD
- 变更说明：<简要描述本次变更内容>
```

### 2. Skills 版本规则

**定义位置**：`specs/meta/skills/skills.md` 及各 Skill 细则文件

**版本递增规则**：

| 变更类型 | 版本动作 | 示例 |
|----------|----------|------|
| 新增 Skill | MINOR++ | `v1.1` → `v1.2` |
| 删除 Skill | MAJOR++ | `v1.x` → `v2.0` |
| 修改 Skill 输入/输出格式 | MAJOR++ | `v1.x` → `v2.0` |
| 新增可选字段（`dependencies`、`quality-metrics`） | MINOR++ | `v1.0` → `v1.1` |
| 修改执行步骤（不改变输出格式） | MINOR++ | `v1.2` → `v1.3` |
| 补充使用场景或约束说明 | MINOR++ | `v1.0` → `v1.1` |

**兼容性声明**：
- Skills 默认与同 MAJOR 版本的 Agents 协作兼容
- 跨 MAJOR 版本变更需在 `agents.md` 中声明兼容性约束

### 3. Tools 版本规则

**定义位置**：`specs/meta/tools/tools.md`、`specs/meta/index/tool-adapters.json`

**版本层次**：

| 层次 | 版本载体 | 递增规则 |
|------|----------|----------|
| 工具链整体 | `tools.md` 元信息 | 按命令集变更递增 |
| 单个命令 | 命令文档/代码注释 | 可独立维护 PATCH |
| 适配清单 | `tool-adapters.json` schema | schema 变更递增 |

**JSON Schema 版本**：
```json
{
  "meta": {
    "schema": "sdd.tool-adapters.v1",
    "created_at": "...",
    "updated_at": "..."
  }
}
```

- Schema 版本格式：`sdd.<artifact>.v<MAJOR>`
- Schema MAJOR 变更表示不兼容的结构调整

**命令版本（可选）**：
- 稳定命令可附加 PATCH 版本：`check-status@1.0.1`
- 一般场景下命令版本省略，以工具链整体版本为准

### 4. 调度契约版本规则

**定义位置**：`specs/meta/index/agent-dispatch.json`

**版本递增规则**：

| 变更类型 | 版本动作 |
|----------|----------|
| 新增/删除 Agent 条目 | MAJOR++ |
| 新增/删除 Skill 条目 | MINOR++ |
| 修改 `skill_owners` 映射 | MINOR++ |
| 修改 `shared_skill_arbitration` 规则 | MAJOR++（若影响流程）或 MINOR++ |
| 修改 `defaults` 配置 | MINOR++ |

**Schema 版本**：
```json
{
  "meta": {
    "schema": "sdd.agent-dispatch.v1",
    "generated_at": "...",
    "source": { ... }
  }
}
```

### 5. 治理与标准文档版本规则

**定义位置**：`specs/govs/`、`specs/standards/`

**版本递增规则**：

| 变更类型 | 版本动作 |
|----------|----------|
| 新增强制要求（MUST） | MINOR++ |
| 删除或弱化强制要求 | MAJOR++ |
| 修改验收标准 | MINOR++ |
| 补充说明或示例 | MINOR++ |
| 修改目标与范围 | MINOR++ 或 MAJOR++（视影响） |

## 版本兼容性矩阵

### 核心文档依赖关系

```
agents.md (v1.x)
    ├── skills.md (v1.x)      # 同 MAJOR 兼容
    ├── agent-dispatch.json   # 由 agents.md 生成，版本跟随
    └── tool-adapters.json    # 引用 agent-dispatch.json

tool-adapters.json (v1)
    └── 各工具入口文件（自动生成）
```

### 兼容性检查规则

1. **同 MAJOR 兼容**：`agents.md v1.x` 与 `skills.md v1.y` 默认兼容
2. **跨 MAJOR 需评审**：任一文档 MAJOR 变更，需检查并更新依赖文档
3. **生成文件跟随**：`agent-dispatch.json`、`tool-adapters.json` 由工具生成，版本由源文档决定

## 版本变更流程

### 常规变更（MINOR++）

1. 修改文档内容
2. 更新元信息中的版本号、最后更新日期、变更说明
3. 如影响索引或契约，运行 `generate-agent-dispatch` 或 `generate-index`
4. 提交评审，记录结论

### 重大变更（MAJOR++）

1. 创建 ADR 说明变更原因与影响范围
2. 更新文档内容与版本号
3. 检查并更新所有依赖文档的兼容性声明
4. 运行完整检查：`check-governance`、`check-completeness`、`check-traceability`
5. 提交评审，获得 Owner 与 reviewer-agent 联签
6. 更新 `CHANGELOG.md`

### 预发布版本流程

1. 使用 `-draft` 或 `-rc<N>` 后缀
2. 完成评审后移除后缀，递增正式版本号
3. 保留预发布版本记录在变更说明中

## 版本记录位置

| 文档类型 | 版本记录位置 |
|----------|--------------|
| Agents/Skills | 文档内 `元信息` 章节 |
| Tools | `tools.md` 元信息 + JSON 文件 `meta` 字段 |
| 治理/标准 | 文档内 `元信息` 章节 |
| 整体变更 | `specs/changelogs/CHANGELOG.md` |

## 强制要求

- MUST-1：所有 SDD 核心文档必须包含 `元信息` 章节，注明版本号、生效日期、最后更新、变更说明。
- MUST-2：MAJOR 版本变更必须创建 ADR 并通过治理评审。
- MUST-3：版本号变更后必须运行对应检查命令验证一致性。
- MUST-4：生成文件（`agent-dispatch.json`、`tool-adapters.json`）不得手动修改版本号，由工具自动维护。

## 检查与验收

- 检查项：
  - 元信息字段完整性
  - 版本号格式合规性
  - 跨文档 MAJOR 版本兼容性
- 验收标准：
  - `check-governance` 通过
  - 依赖文档版本在同 MAJOR 范围内
  - 变更说明准确反映实际变更

## 例外处理
- 紧急修复可跳过 ADR 流程，但需在 3 个工作日内补齐。
- 试验性文档可使用 `-beta` 后缀，不计入正式版本序列。
