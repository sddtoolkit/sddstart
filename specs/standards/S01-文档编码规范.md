## 1. 编码体系概述

### 1.1 编码目标
- 保证文档在整个项目文件夹下的唯一性
- 提供简洁的引用编号供文档间交叉引用
- 支持脚本自动化定位和校验
- 保持文件名语义清晰、可排序

### 1.2 编码组成

文档编码由三部分组成：

```
<前缀>-<编码段>-<SLUG>.md
   ↓       ↓        ↓
 文档类型  定位码    中文语义
```

| 组成部分 | 说明 | 示例 |
|----------|------|------|
| **前缀** | 文档类型标识（大写） | `RQ`, `DS`, `TK`, `ADR`, `G`, `S` |
| **编码段** | 定位码（数字/字母混合） | `10102`, `201260901` |
| **SLUG** | 中文简短描述 | `用户注册`, `API网关认证` |
| **后缀** | 文档类型后缀（可选） | `需求`, `设计`, `任务`, `决策` |

**特殊说明**：G（治理）和 S（标准）类型无前缀分隔符，格式为 `G<NN>-<SLUG>.md` 和 `S<NN>-<SLUG>.md`

### 1.3 分隔符规范

- **唯一分隔符**：`-`（减号/连字符）
- **禁止使用**：下划线`_`、空格、其他特殊符号
- **大小写规则**：前缀必须大写，SLUG使用中文

## 2. 引用编号规则

### 2.1 引用编号定义

**引用编号** = 前缀 + 编码段

引用编号是文档的唯一标识，用于：
- 文档间的交叉引用（如"参见[RQ-10102]"）
- 脚本自动化定位文档
- 追踪矩阵中的关联标识

### 2.2 引用编号唯一性

引用编号在整个 `specs/` 目录下必须唯一，判定规则：

```python
# 唯一性判定逻辑
def get_reference_id(filename):
    """从文件名提取引用编号"""
    # RQ-10102-用户注册需求.md → RQ-10102
    # DS-20101-API网关认证设计.md → DS-20101
    # G01-项目宪章.md → G01
    parts = filename.replace('.md', '').split('-')
    
    if parts[0] in ['RQ', 'DS', 'TK', 'ADR']:
        # 前三段为引用编号（前缀+编码段）
        return '-'.join(parts[:3]) if len(parts) >= 3 else filename
    elif parts[0] in ['G', 'S']:
        # G/S：前缀+NN整体为引用编号
        return '-'.join(parts[:2]) if len(parts) >= 2 else filename
    return filename
```

### 2.3 引用编号冲突处理

当检测到引用编号重复时：
1. **阻断创建**：禁止创建重复编号文档
2. **错误提示**：提示已存在的文档路径
3. **解决方式**：修改编码段或SLUG后重试

| SLUG | 中文简短描述 | `用户注册` |

## 4. SLUG编写规范

### 4.1 基本规则

- **语言**：主要使用中文
- **长度**：4-8个汉字为宜，最长不超过15个汉字
- **内容**：准确概括文档核心主题
- **禁止**：无意义的泛化词汇（如"文档"、"方案"、"最终"）

### 4.2 英文使用规则

仅在以下情况允许使用英文：
1. **专业术语**：Redis、Kubernetes、OAuth2、JWT等
2. **专有名词**：GitHub、Docker、MySQL等
3. **约定俗成**：API、BFF、SaaS、PaaS等

**示例**：
- ✅ `G05-Agent协作宪章.md`（Agent为专业术语）
- ✅ `DS-20101-OAuth2认证设计.md`（OAuth2为标准协议名）
- ❌ `RQ-10101-User注册需求.md`（User可用"用户"替代）

### 4.3 分隔符使用

SLUG内部如需分隔，统一使用`-`：

| 类型 | 示例 |
|------|------|
| 动词+名词 | `用户-注册`, `订单-创建` |
| 限定+主体 | `Vaults-加密解密`, `API-网关认证` |
| 模块+功能 | `支付-回调处理`, `库存-锁定机制` |

## 5. 编码段详细规则

### 5.1 CCC码（3位数字）

CCC码用于对文档进行技术层或业务域的分类。在创建文档时，如果未显式提供 `--ccc` 参数，工具将尝试从**项目简介**中自动推理。

**推理逻辑**：
1. **显式输入**：通过 `--intro "项目简介..."` 参数提供。
2. **基线文档**：自动读取 `specs/govs/G02-项目宪章.md` 或 `specs/1-reqs/requirements.md`。
3. **关键词匹配**：根据简介内容中的技术关键词（如 "前端"、"数据库"、"AI" 等）映射到对应的 CCC 分类。

| 区间 | 用途 | 说明 | 关键词示例 |
|------|------|------|------------|
| 100-199 | 项目核心层 | 项目章程、总体架构、治理规范等 | 架构、治理、规范、宪章 |
| 200-299 | 前端技术层 | Web、移动、桌面、小程序 | 前端、Web、React、Vue、UI |
| 300-399 | 业务领域层 | 用户、订单、支付、商品等 | 用户、支付、订单、业务 |
| 400-499 | 后端技术层 | 微服务、API、消息队列、缓存 | 后端、API、微服务、网关 |
| 500-599 | 数据层 | 数据库、数据仓库、大数据 | 数据、MySQL、Redis、数据库 |
| 600-699 | 组件/工具层 | UI组件、业务组件、工具库 | 组件、工具、库、框架 |
| 700-799 | 专项技术层 | AI/ML、区块链、游戏、嵌入式 | AI、人工智能、区块链 |
| 900-999 | 运维支撑层 | 部署、监控、CI/CD、文档 | 运维、部署、Docker、K8s |

详见附录A《CCC码分类表》

### 5.2 NN码（2位：数字+字母）

**格式**：01-99 → AA-ZZ（排除O、I、L）

**进位规则**：
```
01 → 02 → ... → 09 → 10 → ... → 98 → 99 → AA → AB → ... → AZ → 
BA → BB → ... → BZ → CA → ... → ZZ
```

**可用字母**（23个，排除易混淆的O、I、L）：
`A B C D E F G H J K M N P Q R S T U V W X Y Z`

**最大容量**：99 + 23×23 = 628

### 5.3 YYWW码（4位数字）

- **YY**：年份后两位（00-99）
- **WW**：ISO周数（01-53）
- **示例**：`2609` = 2026年第9周

## 7. 文档操作工具规范

### 7.1 必须通过工具执行CRUD（强制）

所有 `specs/` 目录下的文档操作必须通过 CLI 工具执行，禁止直接手动操作文件。

| 操作 | CLI 命令 | 示例 |
|------|----------|------|
| **创建需求** | `create-requirement` | `sddtool.py create-requirement user-auth` |
| **创建设计** | `create-design` | `sddtool.py create-design api-gateway` |
| **读取文档** | `read-document` | `sddtool.py read-document RQ-10102` |
| **定位文档** | `locate-document` | `sddtool.py locate-document RQ-10102` |
| **检索关联** | `trace-dependencies` | `sddtool.py trace-dependencies RQ-10102` |
| **更换编号** | `rename-document` | `sddtool.py rename-document --by-ref-id RQ-10102 "RQ-10103-新需求.md"` |

#### 禁止直接操作的理由
1. **索引一致性**：工具会自动维护 `meta/index/index.md` 和追溯矩阵
2. **编号唯一性**：工具会自动校验引用编号在整个 `specs/` 目录下的唯一性
3. **SLUG解耦**：通过文档编号定位，SLUG变更不影响定位
4. **审计追踪**：工具操作可记录日志，便于合规审计

#### 违规判定
- 直接手动创建/修改/删除文档视为违反本规范
- 审计发现的手动操作将记录在合规报告中
- 手动操作导致索引不一致的，必须回滚并重新通过工具执行

### 7.2 根据引用编号定位文件

通过文档编号（如 `RQ-10102`）定位文档，**无需携带SLUG信息**。如果同一编号对应多个文档，会提示重复错误。

```python
import os
import re
from pathlib import Path
from typing import Tuple, Optional, List

def locate_document(ref_id: str, specs_dir: str = "specs") -> Tuple[Optional[Path], Optional[str], List[Path]]:
    """
    根据引用编号定位文档文件
    
    Args:
        ref_id: 引用编号，如 "RQ-10102", "G01", "S01"
        specs_dir: specs目录路径
    
    Returns:
        (文档绝对路径, 错误信息, 所有匹配路径列表)
        - 如果找到唯一文档：返回 (path, None, [path])
        - 如果找到多个文档：返回 (None, "错误信息", [path1, path2, ...])
        - 如果未找到：返回 (None, None, [])
    """
    prefix = ref_id.split('-')[0] if '-' in ref_id else ref_id
    
    # 所有类型统一使用 ref_id-* 模式匹配
    if prefix in ['RQ', 'DS', 'ADR', 'TK', 'G', 'S']:
        pattern = f"{ref_id}-*.md"
    else:
        return None, f"未知的前缀类型: {prefix}", []
    
    # 在specs目录下递归搜索
    matches: List[Path] = []
    for root, _, files in os.walk(specs_dir):
        for file in files:
            if re.match(pattern.replace('*', '.*'), file):
                matches.append(Path(root) / file)
    
    if len(matches) == 0:
        return None, None, []
    elif len(matches) == 1:
        return matches[0], None, matches
    else:
        # 找到多个文档，返回错误
        error_msg = f"文档编号 '{ref_id}' 存在重复，找到 {len(matches)} 个文档:\n"
        for i, path in enumerate(matches, 1):
            rel_path = path.relative_to(specs_dir)
            error_msg += f"  {i}. {rel_path}\n"
        error_msg += "请使用 'rename-document' 命令更换其中一个文档的编号"
        return None, error_msg, matches


# 使用示例
if __name__ == "__main__":
    # 定位需求文档 - 只需传入编号，SLUG变更不影响
    path, error, matches = locate_document("RQ-10102")
    if error:
        print(f"错误: {error}")  # 如果有重复会提示详细信息
    elif path:
        print(path)  # 输出: specs/1-reqs/RQ-10102-用户注册需求.md
    else:
        print("未找到文档")
    
    # 定位治理文档
```

### 7.4 提取引用编号

```python
def extract_reference_id(filename: str) -> str | None:
    """
    从文件名提取引用编号
    
    Args:
        filename: 文件名（不含路径）
    
    Returns:
        引用编号，无法提取返回None
    """
    if not filename.endswith('.md'):
        return None
    
    name = filename[:-3]  # 去掉.md
    parts = name.split('-')
    
    if len(parts) < 2:
        return None
    
    prefix = parts[0]
    
    if prefix in ['RQ', 'DS', 'ADR']:
        # RQ-10102-用户注册需求 → RQ-10102
        if len(parts) >= 3 and len(parts[1]) >= 5:
            return f"{parts[0]}-{parts[1][:3]}{parts[1][3:5]}"
        return None
    elif prefix == 'TK':
        # TK-201260901-前端页面开发任务 → TK-201260901
        if len(parts) >= 3 and len(parts[1]) >= 9:
            return f"{parts[0]}-{parts[1][:3]}{parts[1][3:7]}{parts[1][7:9]}"
        return None
    elif prefix in ['G', 'S']:
        # G01-项目宪章 → G01
        # S01-文档编码规范 → S01
        if len(parts) >= 2:
            return f"{parts[0]}-{parts[1]}"
        return None
    
    return None
```
