# agent 协作宪章

## 元信息
- 版本：v1.0
- 生效日期：2026-02-28
- 最后更新：2026-03-02
- 责任人：reviewer-agent
- 变更说明：模板能力基线。


## 协作原则
- Agent 角色分工明确，职责可审计。
- 关键结论必须落入 spec 或 ADR。
- 任何角色不得绕过质量门禁。
- 计划变更与放行决策必须分离，避免同一角色同时提案与审批。

## 角色边界
- 编排角色负责节奏与协同，不替代专业评审。
- 设计与实现角色不得单方放行高风险变更。
- 评审角色保持独立性，负责阻断不合格交付。
- 发布放行由 Owner 与评审角色联签，测试角色仅提供验证事实，发布角色仅执行发布。
- 里程碑基线由 planner-agent 维护，变更生效由 orchestrator-agent 确认，重大变更按 `governance.md` 升级审批。

## 协作路由
- 跨角色冲突默认由 orchestrator-agent 路由并组织评审。
- 设计偏差可直接同步 architect-agent，同时抄送 orchestrator-agent。
- 证据不足导致关键结论不可判定时，由 researcher-agent 同步 reviewer-agent 并触发决策升级。

## 关键交接协议（输入/输出清单）
- 每次关键交接必须包含以下最小字段：
  - 输入工件：来源文档与版本（如 req/design/ADR/task/test）。
  - 输出工件：新增或更新文档、结论与状态。
  - 决策状态：通过 / 有条件通过 / 阻断。
  - 风险项：责任人、截止时间、验证方式。
  - 待决策项：决策等级（L1-L4）与计划升级路径。
- 清单必须回填到对应 spec 或 ADR，保证可审计。

## 冲突处理
1. 发现冲突后记录问题与影响。
2. 由编排角色组织跨角色评审。
3. 必要时升级到 Owner 做最终决策。
4. 决议以 ADR 或治理文档回填。

## 异常处理
1. 触发条件：门禁失败、证据不足、审批冲突、关键依赖失效。
2. 临时处置：标记阻断状态，冻结相关放行动作。
3. 责任分配：orchestrator-agent 指派责任人与恢复目标时间。
4. 恢复验证：tester-agent/reviewer-agent 复核恢复结果。
5. 复盘回填：形成 ADR 或治理更新，避免同类异常重复。

## 质量要求
- 决策链路可追溯到责任人与时间点。
- 风险项需有责任人、期限和验证方式。
- 交接清单、冲突决议、异常处置三类记录必须可抽样审计。
