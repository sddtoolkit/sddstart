- 变更说明：新增操作权限矩阵，明确创建/审批/执行边界。


## 角色清单
- 产品/业务 Owner（人类角色）：目标与验收批准。
- Spec 主编（人类角色）：spec 体系一致性与质量闸门。
- orchestrator-agent：流程编排与跨角色冲突路由。
- specifier-agent：需求澄清与验收口径定义。
- researcher-agent：外部证据检索与归档。
- architect-agent：架构与设计方案。
- planner-agent：任务规划、依赖管理与里程碑基线维护。
- developer-agent：编码与回填。
- tester-agent：验证与质量评估。
- release-agent：发布执行与归档。
- reviewer-agent：独立评审与质量审计。
- 安全/合规负责人：安全与合规审查（默认由 reviewer-agent 兼任；高安全等级项目可拆分独立角色）。

## RACI 约束
- 需求与范围：Owner 负责，Spec 主编审核。
- 架构与关键设计：architect-agent 负责，reviewer-agent 审核。
- 节奏与基线：planner-agent 负责里程碑基线，orchestrator-agent 负责跨角色路由与升级组织。
- 重大变更与发布：Owner 与评审 Agent 联签。
- 发布前验证：测试 Agent 负责验证事实输出，不负责最终放行审批。
- 发布执行：发布 Agent 负责执行与归档，不负责最终放行审批。
- 合规与安全：安全/合规负责人必须参与审批。
- 映射说明：评审 Agent / 测试 Agent / 发布 Agent 分别对应 reviewer-agent / tester-agent / release-agent。

## 操作权限矩阵（简版）
| 工件 | 创建/修改（主责） | 评审/审批 | 执行/推进 |
|---|---|---|---|
| `specs/1-reqs/*` 需求文档 | specifier-agent | reviewer-agent 审查，Owner 进行范围确认 | orchestrator-agent |
| `specs/adrs/*`、`specs/2-designs/*` | architect-agent | reviewer-agent 审查，L3/L4 按治理链路审批 | orchestrator-agent |
| `specs/3-tasks/*` | planner-agent | reviewer-agent 审查，涉及基线变更由 orchestrator-agent 确认生效 | developer-agent、tester-agent |
| `specs/tests/*` 测试计划与结论 | tester-agent | reviewer-agent 审查 | tester-agent |
| `specs/releases/*` 发布记录 | release-agent | reviewer-agent + Owner 联签放行 | release-agent |
| `specs/changelogs/*` 变更日志 | release-agent | reviewer-agent 抽查审计 | release-agent |

## 职责隔离
- 设计者不得单独批准自身设计的重大变更。
- 实现者不得单独批准上线或安全例外。
- 评审者可以参与发布联签，但不得单方放行发布。
- 发布执行人与最终审批人必须分离（release-agent 不得担任最终审批签署角色）。
