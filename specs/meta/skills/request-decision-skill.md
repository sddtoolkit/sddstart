# request decision skill

## 元信息
- 版本：v1.1
- 生效日期：2026-02-28
- 最后更新：2026-03-02
- 变更说明：补齐与 analyze-options-skill 的前置复用约定与路由说明。

## 能力描述
- 当执行流程遇到审批点或多方案分歧时，输出结构化决策请求，降低沟通歧义。

## 使用场景
- 需求范围、技术路线、发布策略存在多个可行选项时。
- Agent 无权限单独决策，需要 Owner 或评审角色确认时。

## 输入格式
- 决策主题、背景、约束条件、候选方案、影响范围。
- 前置 Skill 输出（可选）：`analyze-options-skill` 的方案对比与推荐结论。

## 输出格式
- 决策点标题。
- 背景与影响范围。
- 选项对比表（优点/缺点/风险/推荐度）。
- AI 推荐及理由。
- 需要确认的具体动作。

## 执行步骤
1. 明确当前决策点与触发原因。
2. 若存在 `analyze-options-skill` 输出则直接引用；否则汇总可选方案并标注关键差异。
3. 评估每个方案对工件、风险与交付的影响。
4. 输出标准决策请求并记录待确认项。

## 约束条件
- 本 Skill 仅用于请求与组织决策信息，不拥有审批权。
- 推荐结论必须附带证据来源与边界说明。
- 非 orchestrator-agent 触发时，需通过 orchestrator-agent 路由提交正式决策请求。
