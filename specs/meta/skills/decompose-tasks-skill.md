# decompose tasks skill

## 元信息
- 版本：v1.0
- 生效日期：2026-02-28
- 最后更新：2026-03-02
- 变更说明：模板能力基线。


## 能力描述
- 将需求与设计分解为可执行任务，明确依赖、顺序和验收口径。

## 使用场景
- 设计冻结后进入实现计划阶段。
- 变更影响较大，需要重排任务依赖时。

## 输入格式
- 需求文档、设计文档、相关 ADR、风险清单。

## 输出格式
- 任务分解清单（编号、责任人、依赖、验收标准）。
- 里程碑建议与关键路径提示。

## 执行步骤
1. 识别可独立交付的最小任务单元。
2. 标注依赖关系与并行机会。
3. 定义每项任务的验收标准与回滚要点。
4. 回填任务计划并同步追踪关系。

## 约束条件
- 每项任务必须可验证、可关闭。
- 高风险任务必须显式标注并前置验证。
