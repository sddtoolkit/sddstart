# generate code skill

## 元信息
- 版本：v1.1
- 生效日期：2026-02-28
- 最后更新：2026-03-02
- 变更说明：对齐 check-drift 的来源标记检查要求。

## 能力描述
- 基于需求、设计与任务文档生成可实现的代码骨架与接口占位，确保实现起点与规格一致。

## 使用场景
- 新模块开发时快速建立实现骨架。
- 需要将设计契约转为可执行代码结构时。

## 输入格式
- 需求条目（REQ）。
- 设计文档与接口契约（DSN/API/Schema）。
- 任务分解项（TSK）。

## 输出格式
- 代码目录与文件骨架。
- 关键接口、类型定义与占位实现。
- 与规格关联的注释或标记（`src/` 下 `.ts/.js/.go/.py/.rs` 文件需包含 `GENERATED FROM SPEC`）。

## 执行步骤
1. 提取需求与设计中的实现约束。
2. 映射到模块、接口与数据结构。
3. 生成最小可运行骨架并保留待实现点。
4. 回填任务与追溯关系，进入实现迭代。

## 约束条件
- 本 Skill 生成实现草案，不替代代码评审与发布审批。
- 生成结果必须接受 developer-agent 签收与 reviewer-agent 审核。
- 若输出落在 `src/` 且命中 `check-drift` 扫描后缀，必须写入 `GENERATED FROM SPEC` 以通过一致性检查。
