# generate unit test skill

## 元信息
- 版本：v1.0
- 生效日期：2026-02-28
- 最后更新：2026-03-02
- 变更说明：从通用测试生成能力拆分，明确开发侧单元测试职责。

## 能力描述
- 基于实现代码与设计约束生成单元测试骨架，覆盖核心分支、边界与异常路径。

## 使用场景
- 新增模块实现后快速建立单元测试基线。
- 重构后补齐回归单元测试。

## 输入格式
- 代码变更、函数/模块接口、设计约束、任务条目。

## 输出格式
- 单元测试文件骨架。
- 关键断言与覆盖点清单。
- 与 REQ/DSN/TSK 的追溯备注。

## 执行步骤
1. 提取关键实现路径与失败路径。
2. 生成最小可执行单元测试骨架。
3. 标注待补充断言与测试数据。
4. 回填追溯信息并进入自检流程。

## 执行方式
- 关联检查命令：`python3 specs/meta/tools/sddtool.py check-code-quality`
- 关联校验命令：`python3 specs/meta/tools/sddtool.py check-drift`

## 约束条件
- 本 Skill 仅生成测试骨架，不替代测试结论与发布审批。
- 覆盖缺口必须由 developer-agent 在任务回填中显式记录。
